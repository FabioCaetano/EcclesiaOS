# Feedback De Produto - 2026-05-07

Este documento consolida o feedback de uso real recebido apos a Fase 65. Ele deve orientar as proximas fases antes de novas automacoes grandes.

## Diagnostico Geral

O EcclesiaOS ja possui boa base funcional, mas varios modulos precisam de uma rodada de maturidade operacional. O principal problema atual nao e falta de funcionalidades isoladas, e sim friccao de uso: erros pouco claros, telas muito densas, configuracoes no lugar errado e alguns fluxos publicos quebrando em producao.

## Bloqueadores Imediatos

### Agenda

- Ao criar agenda/evento, a API retornou log bruto/minificado do Prisma em vez de mensagem amigavel.
- Erros de preenchimento precisam apontar exatamente o campo faltante ou invalido.
- O evento foi criado para uma data, mas a recorrencia nao gerou as demais ocorrencias esperadas.
- Exemplo esperado: "todo domingo as 10AM" deveria criar/projetar os proximos domingos, nao apenas o primeiro.

### Rotas Publicas Em Producao

- QR/link de pre-cadastro de visitante retorna `404: NOT_FOUND`.
- Link "Esqueci minha senha" retorna `404: NOT_FOUND`.
- Esses erros indicam problema de roteamento do frontend publicado, provavelmente faltando rewrite/fallback SPA no host.

### Ambientes

- Criar novo ambiente falha se nem todos os campos forem preenchidos, mas o erro nao especifica o campo.
- O erro aparece no formulario de reserva, gerando confusao.
